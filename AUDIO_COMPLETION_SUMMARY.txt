🎓 ARCHITECTURE

  ┌─────────────────────────────────────────────────────────┐
  │  Composants React (WordDetail, etc.)                    │
  │                                                          │
  │  ↓                                                       │
  │  Hook useAudio                                           │
  │                                                          │
  │  ↓  ↓  ↓                                                │
  │  audioService  ← audioRoutes ← audioConfig              │
  │       ↓  ↓  ↓                                           │
  │  Browser API, Fetch API, etc.                           │
  │                                                          │
  │  ↓                                                       │
  │  /audio/{word}.mp3 (ou fallback)                        │
  │                                                          │
  │  ↓                                                       │
  │  Fichiers audio dans public/audio/                      │
  └─────────────────────────────────────────────────────────┘

🔀 HIÉRARCHIE DE FALLBACK INTELLIGENTE

  1. /audio/{word}.mp3     ← Essayé en premier
  2. /audio/{word}.ogg     ← Si MP3 échoue
  3. /audio/{word}.wav     ← Si OGG échoue
  4. /audio/{word}.m4a     ← Si WAV échoue
  5. word.url_prononciation ← Ancien système (fallback final)

  Résultat: Aucun audio n'est perdu, fonctionne TOUJOURS!
